## 《布布与一二的恋爱日记》移动端 Web 应用开发计划

**版本**：v1.0  
**面向角色**：前端开发 
**依据文档**：`docs/移动端WEBAPP需求文档.md`、`docs/ui-design-spec.md`  

---

## 1. 项目目标与范围

- **项目目标**
  - 在uni-app形态下，实现情侣恋爱日记类应用，满足「首页 + 爱的告白 + 恋爱日记 + 爱的相册 + 相爱里程碑」的核心功能，兼顾单手操作、触控体验和视觉质感。
  - 完全本地化数据存储（IndexedDB / localStorage），不依赖后端，可在纯静态托管环境运行。
  - 达成文档中约定的性能与可访问性 KPI：
    - 首屏 TTI ≤ 3s（4G 网络）
    - Lighthouse Performance ≥ 85（移动端模式）
    - Lighthouse Accessibility ≥ 90（含焦点、对比度、语义化）
    - 所有可点击元素触控区域 ≥ 44×44px，无横向滚动
    - 核心功能在弱网环境下保持可用，后续 PWA 阶段支持离线查看核心列表与详情

- **目标用户与使用场景（来自需求文档的补充）**
  - 核心用户：18–35 岁情侣用户，在通勤、睡前、约会后等碎片时间记录恋爱日常。
  - 设备形态：以手机浏览器或「添加到主屏幕后」的 WebView / PWA 为主，单手/双手操作均需友好。
  - 关键体验要点：
    - 入口按钮和主要操作集中在屏幕下半部分，方便拇指点击。
    - 图片上传、预览、拖拽排序等手势交互在移动端流畅自然。
    - 弱网/短暂断网时，至少保证已加载数据可安全回显。

- **范围（按版本阶段）**
  - **MVP（v1.0）**
    - 首页：Slogan + 四宫格入口 + 底部轻导航 / 提醒（满足移动端 TTI 与触控规范）。
    - 爱的告白：文案展示 + 简单交互（随机句子 / 写新的告白）。
    - 恋爱日记：列表、详情查看、新建 / 编辑、基础图片上传与预览、本地存储；支持关键词搜索与基础日期筛选。
    - 爱的相册：创建相册、基础图片上传 / 展示（不含复杂拖拽排序），支持从相册选择或拍照。
  - **v1.1**
    - 相册分类完善、封面设置、收藏、分享（Web Share API 或链接兜底）、相册拖拽排序（长按或排序模式）。
    - 相爱里程碑：时间线、添加 / 编辑、重要时刻标记、基础提醒展示（不含系统通知），首页提醒栏对重要时刻进行显著展示。
  - **v2.0（可选拓展）** 
    - 数据导出 / 导入（JSON / 压缩包）、更完善的纪念日提醒策略。
    - PWA 与离线能力：核心页面和列表/详情的离线访问，支持「添加到主屏幕」后以近似原生的形式使用。

- **非目标（当前版本暂不覆盖）**
  - 账号体系、登录注册、云端同步与多设备数据融合。
  - 复杂的社交分享链路（仅保留基础 Web Share 或链接分享能力）。
  - 服务端数据库与接口层逻辑（当前版本以纯前端本地存储为主，如后续引入需新增独立后端需求文档）。

## 2. 技术选型与架构设计

- **技术栈**
  - 框架：uni-app（H5 形态，目标运行环境为 iOS Safari / Android Chrome 的移动浏览器或 WebView）。
  - 样式与 UI 实现：
    - 全局 CSS 变量实现 `ui-design-spec` 中的色板、字号、间距 token。
    - 使用 Flex / Grid + 媒体查询实现移动端响应式布局。
    - 按需引入轻量动画（CSS transitions / animations）。

- **前端模块划分（代码层面）**
  - `uni-app/main.js` 或等价入口文件：应用入口、全局样式注入、路由/页面配置初始化。
  - `uni-app/pages/`：
    - 首页：Slogan + 四宫格入口。
    - 爱的告白页。
    - 恋爱日记列表 / 详情 / 编辑页。
    - 爱的相册列表 / 详情页。
    - 相爱里程碑时间线 / 编辑页（v1.1）。
  - `uni-app/components/`：
    - 全局组件：`BaseButton`、`BaseCard`、`BaseTextField`、`BottomBar`、`Skeleton` 等。
    - 业务组件：日记卡片、相册卡片、里程碑条目、照片网格、媒体预览弹层等。
  - `uni-app/common/composables/` 或等价目录：
    - `useTheme`（暗色 / 浅色、CSS 变量应用）。
    - `useSafeArea`（safe-area padding 处理）。
    - `useIndexedDB` / `useStorage`（封装 IndexedDB 读写逻辑）。
    - `useMediaPicker`（图片选择 & 压缩）。
  - `uni-app/uni.scss` 或样式目录：
    - 设计 token：颜色、字号、间距等 CSS 变量。
    - 全局 reset、基础排版、全局布局规则与媒体查询。

## 3. 里程碑与时间规划（建议 6–8 周）

> 以下以 1 人前端为基准，可根据实际人力并行调整。

### 3.1 第 1 周：项目初始化与设计落地

- **目标**
  - 完成基础项目骨架、全局设计 token 和 UI 基础组件，为后续页面开发提供统一规范。
  - 对接移动端 WEBAPP 需求文档中的视口、安全区、触控尺寸等通用规范，确保一开始就按移动端标准搭建。
- **主要任务**
  - 初始化 uni-app H5 项目结构，接入 TypeScript（如适用）、ESLint、Prettier。
  - 落地 `ui-design-spec`：
    - 全局 CSS 变量（颜色、字号、spacing）。
    - 全局布局：视口 meta、safe-area、基础栅格 / 间距系统。
  - 开发通用组件 v1：
    - `PrimaryButton`、`SecondaryButton`、`IconButton`。
    - `Card` 基础样式。
    - `TextField`、`TextArea` 表单组件。
    - `Tag`、`Badge`。
  - 搭建路由骨架与空页面（含基本布局）：
    - `/`、`/diary`、`/diary/:id`、`/album`、`/album/:id`、`/confession`、`/milestone`。
- **验收**
  - 项目可跑通，路由切换正常。
  - 通用组件视觉与交互符合 `ui-design-spec.md` 的基础规范。

### 3.2 第 2–3 周：MVP 功能（首页 + 爱的告白 + 恋爱日记）

- **目标**
  - 实现 MVP 中「首页、爱的告白、恋爱日记」的完整动线与本地存储。
- **主要任务**
  - 首页
    - 实现 Slogan 区、四宫格入口、底部提醒 / 设置入口。
    - 确保四宫格触控区域、动效（轻微缩放 + 阴影变化）、徽标 / Badge 状态。
    - 验证首页在 4G 网络下首屏 TTI ≤ 3s，无横向滚动，主要操作集中在可拇指触达区域。
  - 爱的告白
    - 页面结构、主卡片文案展示。
    - 底部固定操作区（粘底，考虑 safe-area）。
    - 随机告白文案的简单实现（本地数组或 JSON）。
    - 勾连「写新的告白」与「保存到日记」入口。
  - 恋爱日记
    - 数据模型设计：`diaries` Object Store 字段（ID、标题、内容、日期、媒体引用、隐私状态等）。
    - 列表页：
      - 顶部搜索与筛选（关键词 + 日期 + 私密 / 公开）。
      - 列表项 UI（标题、摘要、日期、媒体缩略图 / 私密图标）。
    - 详情 / 编辑页：
      - 标题、日期、内容输入区。
      - 图片上传 & 缩略图展示（含删除、点击预览）。
      - 隐私切换（私密 / 公开）。
      - 底部工具栏（图标按钮 + 保存）。
    - IndexedDB 操作封装：
      - 初始化数据库。
      - 新增 / 更新 / 删除 / 查询日记。
    - 搜索与筛选：
      - 关键词搜索栏与日期筛选条。
      - 筛选私密 / 公开状态，交互形式与 UI 规范中的 Segmented Control 一致。
- **验收**
  - 在真机（iOS Safari、Android Chrome）上测试：
    - 首页 → 日记列表 → 新建 → 保存 → 列表 / 详情展示全流程可用。
    - 触控区域满足 ≥ 44×44px。
  - Lighthouse（Mobile 模式）：
    - Performance ≥ 75（MVP 阶段，可逐步优化到 85+）。
    - Accessibility ≥ 85。

### 3.3 第 4–5 周：爱的相册 + 相爱里程碑（v1.1）

- **目标**
  - 完成相册与里程碑模块，覆盖主要功能点（不含 PWA）。
  - 对齐需求文档中的用户故事：支持通过时间线回顾重要时刻，并能在首页获得重要里程碑提醒。
- **主要任务**
  - 爱的相册
    - 数据模型：`albums`、`photos` Object Store 字段设计。
    - 相册列表页：
      - 双列卡片布局、封面展示、照片数量、收藏标记。
      - 长按进入排序 / 多选模式（可先实现多选删除，再迭代拖拽排序）。
    - 相册详情页：
      - 大封面 + 照片网格（3 列）。
      - 多选 / 拖拽排序（如拖拽较复杂，可拆到后续迭代）。
      - 上传流程：选择图片 / 拍照 → 压缩 / 校验 → 写入 IndexedDB → UI 刷新。
      - 权限处理：对于相册/相机权限拒绝场景，给出用途说明和「去设置开启」指引，保证流程可感知、可恢复。
  - 相爱里程碑
    - 数据模型：`milestones` Object Store。
    - 时间线页：
      - 垂直时间线 + 重要时刻高亮 + 卡片样式。
    - 新建 / 编辑页：
      - 表单字段顺序与 UI 规范保持一致。
      - 日期选择器封装（底部弹出式）。
      - 重要时刻切换控件。
- **验收**
  - iOS / Android 真机测试：
    - 相册上传、滚动性能、图片预览流畅。
    - 时间线滚动、卡片展示正常。
  - Lighthouse：
    - Performance ≥ 80。
    - Accessibility ≥ 90。

### 3.4 第 6 周及以后：性能优化 + PWA + 数据导出（v2.0）

- **目标**
  - 达成最终 KPI，补齐 PWA 和数据持久化增强能力。
- **主要任务**
  - 性能优化
    - 首屏资源拆分与懒加载（路由按需加载、组件按需加载）。
    - 图片懒加载和适配尺寸（`loading="lazy"` 等）。
    - 骨架屏：日记列表、相册列表加载时 skeleton。
  - 可访问性优化
    - ARIA 标签、语义化标签（`main`、`header`、`nav`、`section`）。
    - 焦点顺序、键盘可操作性（至少保证基础可聚焦）。
    - 对比度检查与微调。
  - PWA 与离线能力
    - 编写 manifest（名称、图标、主题色）。
    - 注册 Service Worker，缓存静态资源与关键页面数据。
    - 离线模式下的 UI 提示与兜底处理：
      - 已缓存的日记列表、日记详情、相册列表、相册详情、相爱里程碑时间线在离线时仍可访问。
      - 未缓存数据的访问请求给出明显但温和的提示（如「当前离线，仅展示本地缓存内容」）。
  - 数据导出 / 导入
    - 导出：将 IndexedDB 中的核心数据导出为 JSON / 压缩包。
    - 导入：从文件恢复数据，提供「覆盖 / 合并」策略说明和确认弹窗。
- **验收**
  - Lighthouse（Mobile，PWA 检查通过）：
    - Performance ≥ 85。
    - Accessibility ≥ 90。
  - 核心页面在离线状态下可正常打开并查看本地已保存的数据列表 / 详情。

## 4. 详细任务拆解（按模块）

### 4.1 基础设施 & 工程化

- 工程搭建
  - 创建 uni-app H5 工程，若条件允许则接入 TS 支持。
  - 配置 ESLint + Prettier。
  - 配置路径别名、环境变量。
- 样式系统
  - 实现 `tokens.css`：颜色变量、字体层级、spacing token。
  - 全局 reset + 基本 typography。
  - 响应式规则与媒体查询封装。

### 4.2 全局组件

- 开发并沉淀：
  - `Button` 系列组件（Primary / Secondary / Icon）。
  - `Card`、`Modal` / `BottomSheet`。
  - `Input` / `TextArea` / `Toggle` / `SegmentedControl`。
  - `Tag`、`Badge`、`Skeleton`。
- 要求：
  - 触控反馈（scale + 阴影动画）。
  - 可访问性属性（aria-label 等）。
  - 统一尺寸规范（≥ 44×44px 触控区域）。

### 4.3 页面与业务逻辑

- 首页
  - 静态布局 → 动效 & 状态（未读 / 里程碑提醒）。
  - 骨架屏与加载状态设计，避免弱网下长时间白屏。
- 爱的告白
  - 文案源（本地 JSON / TS 常量）、随机策略。
  - 与日记模块的联动：一键保存为新日记草稿。
- 恋爱日记
  - CRUD 逻辑、过滤与搜索。
  - 媒体预览组件复用给相册。
- 爱的相册
  - 照片上传限制与压缩策略。
  - 网格布局 & 滚动性能调优。
- 相爱里程碑
  - 时间线绘制组件。
  - 重要时刻的样式与提醒逻辑（首页提醒条）。

### 4.4 数据与存储封装

- IndexedDB 封装
  - 初始化、版本管理。
  - 标准化的 CRUD 接口。
  - 错误处理与降级策略。
  - 媒体存储策略：
    - 小体积图片可直接以 base64/blob 形式存入 IndexedDB。
    - 对大体积图片做压缩与数量控制，必要时在 UI 中提示存储空间风险。
- localStorage
  - 封装 `useLocalStorage` composable。
  - 主题、最近访问页面等轻量状态。

## 5. 风险与应对策略

- 弱网 & 首屏性能
  - 提前规划资源体积（图片压缩、依赖控制）。
  - 骨架屏 + 渐进加载，避免白屏。
  - 对首页和列表页进行 4G 真机测试，校准 TTI 和交互流畅度。
- 不同机型兼容性
  - iOS Safari 与 Android Chrome 真机穿插测试。
  - safe-area、键盘弹起场景重点验证。
- 浏览器存储限制
  - 控制单张图片大小与数量，必要时增加提示与清理入口。
- 交互复杂度（拖拽排序、PWA）
  - 拖拽排序可后置到 v1.1 或 v2.0。
  - PWA 在主功能稳定后再集成，以降低调试和发布成本。

## 7. 详细执行步骤清单（可勾选）

> 说明：以下步骤基于单人前端开发假设，按照「先基础设施 → 再核心功能 → 再扩展模块 → 最后优化与 PWA」的顺序拆解。实际执行时可根据进度和优先级微调顺序，但建议保持阶段内的依赖顺序不变。

### 7.1 环境搭建与基础设施（第 1 周）

1. 安装/确认开发环境：Node、HBuilderX 或 CLI、移动端调试工具（Chrome DevTools、Safari Web Inspector）。
2. 创建 uni-app H5 工程，配置项目名称、包名、基本目录结构。
3. 集成 ESLint + Prettier（或等价工具），配置基础规则（含 Vue/uni-app 支持）。
4. 配置路径别名（如 `@/` 指向 `uni-app/`）、环境变量文件（开发/生产）。
5. 在 `uni.scss` 中实现基础设计 token：颜色、字号、spacing 变量，与 `ui-design-spec.md` 对齐。
6. 落地全局样式：reset、基础排版（body 字体、背景色）、全局布局容器。
7. 在入口文件中配置 `<meta viewport>` 与 `viewport-fit=cover`，并验证 safe-area 生效。
8. 搭建基础页面结构：在 `pages.json` 中配置首页、日记列表/详情/编辑、相册列表/详情、告白、里程碑页面路由。
9. 为每个页面创建对应的 `.vue` 文件，放入基础布局骨架（顶部标题区、主内容区、底部操作区）。
10. 开发基础 UI 组件 v1：`BaseButton`、`BaseCard`、`BaseTextField`、`Tag`、`Badge`，并在一个 demo 页面中联调样式和交互。
11. 在真机上打开 demo，检查 safe-area、字体、主色系是否符合视觉预期。

### 7.2 MVP 功能实现：首页 + 爱的告白 + 恋爱日记（第 2–3 周）

**首页**
