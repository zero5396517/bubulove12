## 《布布与一二的恋爱日记》移动端 Web 应用 UI 设计规范

- **版本**：v1.0  

- **对应文档**：`docs/移动端WEBAPP需求文档.md`（移动端 WEBAPP 需求与技术规范）

---

## 1. 整体设计风格

- **整体基调**：温柔、恋爱感、偏少女系，但不过度卡通；强调“私密日记 + 仪式感”。
- **视觉方向**：软糯渐变 + 卡片式布局 + 圆角与轻拟物阴影。
- **操作方式**：移动端优先，单手拇指操作友好，主要操作区聚焦在屏幕下半部分。

---

## 2. 设计规范总览

### 2.1 颜色系统（CSS 变量）

- `--color-primary: #FF7A9E;` 主色（布布粉），用于主按钮、高亮态。
- `--color-primary-gradient: linear-gradient(135deg, #FF7A9E 0%, #FFB0C9 50%, #FFC8D6 100%);` 主色渐变。
- `--color-secondary: #B49CFF;` 辅色 1（柔和紫，用于次要按钮、信息点缀）。
- `--color-accent: #FFD27F;` 辅色 2（温暖黄，用于里程碑、提醒）。
- `--color-bg: #FFF6FA;` 应用主背景色。
- `--color-card-bg: #FFFFFF;` 卡片背景色。
- `--color-text-main: #3F2A35;` 主文本颜色。
- `--color-text-sub: #8C6F80;` 次文本颜色。
- `--color-border-subtle: rgba(63,42,53,0.08);` 轻分割线/边框。
- `--color-danger: #FF4D6A;` 删除/危险行为色。
- `--color-overlay: rgba(26,13,23,0.55);` 蒙层背景，用于图片预览等。

> 对比度要求：主文本与背景对比度 ≥ 4.5:1；按钮文字与背景 ≥ 4.5:1。
> 当用户处于强光/暗光环境时，仍能清晰阅读主要内容与操作文案。

### 2.3 触控与可访问性（移动端要求对齐需求文档）

- 触控尺寸：所有可点击/可触摸区域物理尺寸建议 ≥ 44×44px，可通过外层增加点击热区实现。
- 触控反馈：按钮、卡片、图标等交互元素在点击时必须有明显的视觉反馈（缩放、阴影变化或背景变化）。
- 滚动行为：
  - 竖向滚动为主，任何页面禁止出现无意义的横向滚动条。
  - 图片网格、卡片列表在小屏下也应保持清晰可见且易于操作。
- 键盘与输入：
  - 输入区域获得焦点时，键盘弹起后重要按钮（如“保存”）不得被完全遮挡。
  - 底部工具栏与键盘交互时，使用安全区和适当的 `padding-bottom` 处理。

### 2.2 字体与字号

- **字体族**：
  - 主字体：`"SF Pro Rounded", "PingFang SC", "Helvetica Neue", Arial, sans-serif;`
  - 英文字体建议使用略带圆角的家族，营造恋爱氛围。

- **字号层级（移动端）**：
  - 一级标题（页面主标题）：`24px` / 行高 `32px` / 字重 `600`
  - 二级标题（模块标题）：`18px` / 行高 `26px` / 字重 `600`
  - 正文主文案：`15px` / 行高 `22px` / 字重 `400`
  - 说明/辅助文字：`13px` / 行高 `18px` / 字重 `400`
  - Tab / 按钮文字：`14px` / 行高 `20px` / 字重 `500`
  - Tag / 徽标：`12px` / 行高 `16px` / 字重 `500`

## 3. 布局与栅格

### 3.1 视口与安全区

- 视口：`<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">`
- 内容宽度：
  - 以 375px 设计稿为基准，左右内边距：`16px`。
  - 卡片内容区域最大宽度：`375px - 2 * 16px = 343px`。
- 安全区域：
  - 头部/底部使用：`padding-top: env(safe-area-inset-top);`
  - 底部导航：`padding-bottom: max(env(safe-area-inset-bottom), 12px);`

### 3.2 间距系统

- 全局间距 token：
  - XS：`4px`（图标与文字微间距）
  - S：`8px`（按钮内元素间距、小模块间距）
  - M：`12px`（列表项内部上下间距）
  - L：`16px`（模块之间间距、卡片内 padding）
  - XL：`24px`（大区块顶部留白、页面首段留白）

- 列表项（如日记列表行）：
  - 最小高度：`64px`
  - 左右内边距：`16px`
  - 行内元素间距：`8–12px`

## 4. 全局组件规范

### 4.1 按钮（Button）

**主按钮（Primary）**

- 高度：`44px`
- 最小宽度：`min(100%, 320px)`
- 背景：`var(--color-primary-gradient)`
- 圆角：`999px`（胶囊形）
- 文字：`14px / 500`，颜色 `#FFFFFF`
- 阴影：`0 10px 20px rgba(255,122,158,0.25)`
- 点击态：整体 `scale(0.97)`，阴影减弱，背景亮度降低 5–10%
- 禁用态：`opacity: 0.5; pointer-events: none;`

**次按钮（Secondary）**

- 高度：`40px`
- 边框：`1px solid rgba(255,122,158,0.4)`
- 背景：`rgba(255,250,252,0.85)`
- 圆角：`999px`
- 相对于主按钮权重更低，可放在主按钮下方或右侧。

**图标按钮（Icon Button）**

- 尺寸：`40×40px`
- 用途：浮动添加、相册上传、录音等主要动作。
- 触控区域：最小 `44×44px`，可通过外层 `hit-area` 增大。

### 4.2 卡片（Card）

**基础卡片**（首页入口/日记卡片等）

- 宽度：100%（减去页面左右内边距）
- 内边距：`16px`
- 圆角：`18px`
- 背景：`var(--color-card-bg)`
- 阴影：`0 8px 24px rgba(43,21,33,0.06)`
- 边框：`1px solid rgba(255,255,255,0.65)`（营造玻璃感）
- 交互：轻微 hover/press 动画：`transform: translateY(-1px)` 或 `scale(0.99)`

### 4.3 输入与表单

**输入框（TextField / TextArea）**

- 高度（单行）：`44px`
- 多行：最小高度 `96px`，支持垂直扩展。
- 圆角：`12px`
- 背景：`rgba(255,255,255,0.9)`
- 边框：
  - 默认：`1px solid rgba(140,111,128,0.25)`
  - 聚焦：`1px solid #FF7A9E` + `box-shadow: 0 0 0 2px rgba(255,122,158,0.12)`
- 内边距：`10px 12px`
- 占位符：颜色 `rgba(63,42,53,0.35)`

### 4.4 标签与徽标

**Tag（如「重要时刻」「私密」）**

- 高度：`20px`
- 圆角：`999px`
- 背景：
  - 重要时刻：`rgba(255,210,127,0.25)`，文字 `#AA6B11`
  - 私密：`rgba(255,122,158,0.18)`，文字 `#FF4D6A`
- 内边距：`0 8px`
- 字号：`12px`，字重 `500`

**数字徽标（Badge）**

- 圆形或胶囊：
  - 尺寸：`18×18px`（单数字）或高度 `18px`，内边距 `0 6px`
  - 背景：`#FF4D6A`
  - 文本：`#FFFFFF`，`11px`

## 5. 页面级 UI 规范

### 5.1 首页（Slogan + 四宫格入口）

**结构**

- 顶部：柔和渐变背景 + Slogan + 简单插画/心形图案。
- 中间：四宫格入口卡片（恋爱日记 / 爱的相册 / 爱的告白 / 相爱里程碑）。
- 底部：导航/设置入口 + 今日提醒/里程碑提示条。

**Slogan 区**

- 高度：视口高度约 35–40%
- 文案样式：
  - 主文案：`24px` / 600 / `var(--color-text-main)`
  - 副文案：`14px` / 400 / `var(--color-text-sub)`，行宽控制在 2–3 行。
- 对齐：垂直居中，左右居中或轻微偏左。
- 背景：柔和渐变 + 小爱心、星星等 SVG 装饰。

**入口按钮（四宫格）**

- 布局：2 行 × 2 列。
- 每个入口尺寸：
  - 高度：`96–112px`
  - 内间距：`12–14px`
  - 圆角：`18px`
- 每个入口内容：
  - 顶部图标：`32×32px`（线性或双色图标）
  - 标题：`15px` / 500
  - 副文本（可选）：`12px` / `var(--color-text-sub)`
- 触控区域：整体卡片 `≥ 44×44px`，卡片间水平间距 `12px`，垂直间距 `12–16px`。

**状态与交互**

- 轻触：卡片 `scale(0.97)` + 阴影略减。
- 常驻状态标识：
  - 有未读或今日任务时，在入口右上角展示红色 `Badge`。
  - 入口卡片下方可有微小进度条（例如今日完成度）。

### 5.2 爱的告白页面

**结构**

- 顶部：返回按钮 + 标题「爱的告白」。
- 主区：大面积渐变卡片，展示固定文案/可编辑长文本。
- 底部：操作按钮（如“随机一句告白”、“写一封新的告白信”）。

**主卡片**

- 距顶部间距：`24px`
- 宽度：全宽，左右内边距 `16px`
- 内卡片：
  - 圆角：`20px`
  - 背景：粉色 / 紫色渐变 + 轻噪点纹理（可用 CSS 背景图）
  - 内边距：`20–24px`
- 文案段落：
  - 主句：`20px` / 600，居中
  - 副句：`14px` / 400，行高 `22px`，上下间距 `8–12px`

**底部操作区**

- 两个主次按钮：
  - 主按钮：`“写一封新的告白”`
  - 次按钮：`“换一句告白”或“保存到日记”`
- 使用页面固定底栏：`position: sticky; bottom: 0; background: linear-gradient(...); padding: 12px 16px max(env(safe-area-inset-bottom), 16px);`

### 5.3 恋爱日记

#### 5.3.1 日记列表页

**顶部**

- 标题「恋爱日记」。
- 右侧图标按钮：搜索、筛选（两个 24px 图标按钮）。

**筛选条**

- 日期选择按钮：`“全部日期”`（带日历图标）。
- 筛选状态按钮：`“全部 / 仅私密 / 仅公开”`（使用 Segmented Control）。
- 高度：`40px`，圆角 `999px`，放在浅色卡片背景上。
  - 关键词搜索框：可固定在列表顶部或采用搜索图标点击展开，下方使用轻量阴影提升层级。

**列表项样式**

- 高度：最小 `80px`，可随内容扩展。
- 布局：
  - 左侧：日期 + 小圆点轴（可选）。
  - 中间：标题（单行或两行截断）+ 摘要内容（2 行以内）。
  - 右侧：封面图缩略图（若存在多媒体）或私密图标。
- 文本：
  - 标题：`15px` / 500，最多两行，超出省略。
  - 摘要：`13px` / 400，颜色 `var(--color-text-sub)`。
- 底部分割线：`1px solid rgba(63,42,53,0.06)` 或使用卡片间距代替。

#### 5.3.2 日记详情/编辑页

**头部区**

- 标题输入框（编辑模式）：大号输入 `20px`，单行，聚焦突出边框。
- 日期展示/选择：`14px` 文本，附日期选择图标。

**内容区**

- 文本输入区域：
  - 多行 TextArea，圆角 `16px`，背景为浅白。
  - 支持 emoji、段落空行；行高 `22px`。
- 媒体区域：
  - 图片缩略图为 3 列网格：
    - 每个缩略图宽高约 `100×100px`
    - 圆角 `12px`
    - 右上角显示删除/私密标识
  - 点击后进入全屏预览：
    - 背景黑透明 `var(--color-overlay)`
    - 支持左右滑动切换图片
    - 顶部关闭按钮（X）与“设为封面”等操作。

**底部工具栏**

- 固定在键盘上方/页面底部：
  - 图标从左到右：添加图片（相册）、拍照、录音、隐私切换（私密/公开）、保存按钮。
  - 每个图标按钮 `40×40px`，触控区域 `≥ 44×44px`。
- 保存按钮为主色小胶囊按钮（如文字“保存”）。
 - 工具栏在键盘弹起时跟随上移，确保保存按钮始终在视口内可见。

### 5.4 爱的相册

#### 5.4.1 相册列表页

**卡片布局**

- 两列瀑布或规则网格：
  - 每个相册卡片宽度：`(100% - 16px - 8px) / 2`（左右 16px padding，中间 8px gutter）。
  - 高度：约 `180px`（上图下文）。
- 内容：
  - 上方为封面图（占约 70% 高度），圆角 `18px`。
  - 下方为相册名 + 照片数量 + 收藏标记。
- 长按卡片进入“排序 / 多选模式”，顶部出现操作栏（取消/完成、全选、删除）。

#### 5.4.2 相册详情页（单个相册）

**顶部**

- 大封面（宽度全屏，高度约 `220px`，圆角可仅底部 `24px`）。

**照片网格**

- 3 列网格：
  - 单元格：`(100% - 16px * 2 - 4px * 2) / 3`
  - 圆角：`12px`
- 支持多选（勾选蒙层）、拖拽排序（长按进入排序模式）。
 - 上传中的图片使用骨架方块和进度指示，失败状态需以小图标+提示文案标明，可重新上传。

**底部操作区**

- 左侧次按钮：“从相册选择”、“拍照”。
- 右侧主按钮：“上传到这个相册”。
- 上传过程展示顶部进度条和缩略图 loading 状态（骨架方块）。

### 5.5 相爱里程碑

**时间线页面**

- 顶部：标题「相爱里程碑」+ 添加按钮（+）。
- 内容：
  - 垂直时间线，左侧为日期点和竖线，右侧为里程碑卡片。
  - 时间点圆点大小：`10px`，重要时刻使用渐变边框或发光效果。
- 里程碑卡片：
  - 宽度：`100% - 32px`（为了留出时间线）。
  - 圆角：`16px`。
  - 内部：
    - 标题：`15px` / 600。
    - 日期：`13px` / `var(--color-text-sub)`。
    - 说明：`13px`，最多 3 行截断。
    - 媒体缩略：若有图/语音，以小图标或图片显示。

**新建/编辑表单**

- 字段顺序：名称（必填）→ 日期（必填）→ 是否重要时刻 → 文本说明 → 媒体上传。
- 日期选择器：优先使用原生 `<input type="date">`，封装成弹出式底部动作面板。
- “是否重要时刻”使用切换开关（Toggle）或 Tag 按钮。

## 6. 交互与动效规范

- 点击反馈：
  - 所有可点击元素触发时，提供视觉反馈：
    - 按钮：`scale(0.97)` + 阴影减弱。
    - 卡片：轻微缩放或背景加深。
  - 过渡时间：`120–180ms`，缓动函数 `cubic-bezier(0.22, 0.61, 0.36, 1)`。

- 页面切换：
  - 使用淡入 + 轻微上移动画（如 `translateY(8px) → 0`）。
  - 动画时间：`200–250ms`，低端机需保证流畅。

- 骨架屏/加载状态：
  - 列表、卡片加载时展示浅粉色/浅灰 skeleton。
  - 骨架圆角与真实卡片一致。

- 空状态与错误状态：
  - 列表为空时使用插画 + 简短文案（例如「还没有记录，今天想和 TA 写点什么吗？」）。
  - 网络错误或离线时，在顶部展示轻提示条（浅色背景 + 图标 + 文案），说明当前为离线/异常状态，并弱化主要操作按钮，避免误操作。

## 7. 适配与响应式参数

- 设计基准：375×667。
- 适配区间：
  - 320–360 宽度：卡片内边距从 `16px` 缩减为 `12px`，Slogan 字号减小 2px。
  - ≥ 414 宽度：卡片内边距可增至 `20px`，Slogan 字号增大 1–2px。
- 媒体查询建议：
  - `< 360px`：`font-size` 全局缩放 `0.95`。
  - `≥ 414px`：主要标题、Slogan 增加 `1–2px`。

## 9. PWA / 添加到主屏幕相关视觉

- 应用图标
  - 形状：圆角矩形或略带圆形的心形图案，避免过多细节，保证在 48×48 以上尺寸下仍然清晰。
  - 颜色：使用主色渐变 `var(--color-primary-gradient)` 作为背景，中心可放简化的「日记本 + 心形」符号。
- 启动画面（Splash）
  - 背景色：与 `--color-bg` 或主渐变保持一致，避免冷启动时明显的闪屏。
  - 内容：应用图标居中，下方一行简短文案（如「和 TA 的点滴，都在这里」）。
  - 文案字号：约 `16px`，颜色使用 `var(--color-text-sub)`。
- 全屏/独立窗口样式
  - 状态栏与导航栏主题色与应用主色系一致。
  - 顶部/底部安全区仍需保留 `env(safe-area-inset-*)` 处理，保证在刘海屏设备上的体验一致。
