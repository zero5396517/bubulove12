# 移动端 WEBAPP 需求文档

**项目名称**：《布布与一二的恋爱日记》uni-app 应用  
**文档类型**：移动端 WEBAPP 产品需求与技术规范（纯前端版）  
**版本**：1.0  
**基于**：产品需求文档（PRD）v1.0  
**日期**：2025年3月  

---

## 一、执行摘要

### 1.1 问题陈述

目标用户（18–35 岁情侣）需要在手机上随时记录恋爱日常、相册与里程碑，现有 PRD 定义了功能范围，但缺少针对uni-app移动端  的体验标准与技术约束，易导致触控、性能、离线与多端一致性等问题。

### 1.2 解决方案

将《布布与一二的恋爱日记》以**移动端优先（Mobile First）** 的纯前端 Web 应用形式交付：仅依赖浏览器端能力（如 uni-app 、localStorage、Cache Storage 等）进行数据存储与读取， 并在性能、安全与可访问性上设定可量化指标。

### 1.3 成功标准（可衡量 KPI）

| KPI | 目标值 | 说明 |
|-----|--------|------|
| 首屏可交互时间（TTI） | ≤ 3s（4G 网络） | 在常规 4G 环境下测量 |
| Lighthouse 性能分数 | ≥ 85 | 移动端模式 |
| Lighthouse 可访问性分数 | ≥ 90 | 含焦点、对比度、语义化 |
| 触控目标尺寸 | ≥ 44×44px | 所有可点击/可触摸区域 |
| 核心功能离线可用 | 可选（PWA 阶段） | 日记列表、相册列表等可缓存查看 |

## 二、目标用户与使用场景

### 2.1 用户画像

- **核心用户**：18–35 岁，对恋爱记录、情感留存、轻度互动有需求的用户。
- **设备**：以手机为主（iOS / Android），浏览器或「添加到主屏幕」后的 WebView/PWA。
- **场景**：通勤、睡前、约会后等碎片时间，单手或双手操作。

### 2.2 用户故事（移动端侧重）

| 角色 | 故事 | 验收要点（移动端） |
|------|------|--------------------|
| 情侣用户 | 作为用户，我希望在手机上快速打开应用并看到首页，以便马上选择功能 | 首屏 3s 内可操作；无横向滚动；重要按钮≥44px |
| 情侣用户 | 作为用户，我想用拇指轻松点击「恋爱日记」「爱的相册」等入口 | 两行两列按钮触控区域≥44×44px；有点击态反馈 |
| 情侣用户 | 作为用户，我想在日记里上传相册中的多张图片并预览 | 支持多选、压缩/缩略图、点击放大、手势关闭 |
| 情侣用户 | 作为用户，我想在相册里拖拽排序或设置封面 | 支持触摸拖拽；长按或菜单设置封面 |
| 情侣用户 | 作为用户，我想在弱网或短暂断网时仍能查看已加载的日记/相册 | PWA 阶段：核心列表与详情可离线查看 |

## 三、移动端功能需求（按模块）

### 3.1 通用移动端规范

- **视口与布局**
  - 使用标准 viewport meta，禁止用户缩放时可考虑 `user-scalable=no`（需评估无障碍后再定）。
  - 安全区域：预留 `env(safe-area-inset-*)`，适配刘海屏与底部横条。
- **触控与反馈**
  - 所有可点击/可触摸元素触控区域 ≥ 44×44px，可视觉上小于 44px 但点击热区需足够。
  - 点击有即时反馈（如缩放、高亮、loading），避免 300ms 延迟（使用 `touch-action` 等）。
- **输入与键盘**
  - 输入框获得焦点时自动弹起合适键盘（如日记内容用多行文本、日期用日期选择器）。
  - 键盘弹起时页面可滚动，重要按钮不被完全遮挡。

### 3.2 首页

- **Slogan 展示**
  - 全屏标语，字体居中，颜色柔和（如粉色、浅紫）；移动端字号需在 18px 以上，保证可读性。
- **入口按钮**
  - 两行两列，四个入口（与 PRD 一致）；按钮/触控区域 ≥ 44×44px，间距均匀；支持点击动效（如 scale）。
- **交互与状态**
  - 点击跳转对应页面；返回首页时恢复原界面状态（如未读/红点）。

### 3.3 爱的告白

- 用于布布向一二表达爱意；移动端需保证文案可读、按钮易点，无横向溢出。

### 3.4 恋爱日记

- **新建/编辑**
  - 标题、内容（文字、表情、段落格式）、多媒体（图片、语音多选）、隐私（私密/公开）。
  - 移动端：图片支持多选、压缩或缩略图上传、点击预览大图并可手势关闭；语音支持录制与播放条。
- **列表与查看**
  - 列表项触控区域足够；支持下拉刷新（若有列表接口）；详情页图片/语音可点击播放。
- **搜索与筛选**
  - 关键词搜索、按年/月/日筛选；移动端提供日期选择器（原生或 UI 组件），筛选项易点选。

### 3.5 爱的相册

- **上传**
  - 从相册选择或拍照，多张上传；移动端需请求相册/相机权限并做降级提示。
- **分类与排序**
  - 创建相册、拖拽排序；移动端支持长按拖拽或排序菜单。
- **封面与收藏**
  - 设置/删除封面；收藏与「收藏夹」展示；分享到微信、微博等（调用 Web Share API 或兜底链接）。

### 3.6 相爱里程碑

- **添加/编辑**
  - 名称、日期（支持农历/公历）、说明、媒体、是否「重要时刻」。
  - 移动端日期选择器友好；重要时刻在首页提醒栏展示。
- **时间线**
  - 按时间倒序；列表项可点击查看详情；纪念日支持提前 N 天提醒（本地或服务端提醒）。

## 四、技术规格（移动端 WEBAPP）

### 4.1 架构与栈 

- uni-app   

### 4.2 前端存储与数据模型

- **总体原则**：所有业务数据（恋爱日记、相册、照片、里程碑等）仅存储在浏览器侧，不依赖服务器数据库；不同设备之间不做自动同步（除非后续引入登录与同步方案，此处视为非目标）。  
- **推荐存储方案**：
  - 结构化数据（如日记、相册、里程碑记录）使用 IndexedDB（可通过封装或第三方轻量库简化操作）。
  - 轻量配置与 UI 偏好（如主题、最近打开页面）可使用 localStorage。
  - 若启用 PWA，静态资源与部分列表数据可用 Cache Storage 进行缓存。
- **示例数据表（IndexedDB 中的 Object Store）**：
  - `diaries`：日记记录（ID、标题、内容、媒体引用、时间、隐私状态等）。
  - `albums`：相册（ID、名称、封面引用、标签、创建时间等）。
  - `photos`：照片（ID、所属相册 ID、文件引用、标签、时间、是否收藏等）。
  - `milestones`：里程碑（ID、名称、日期、内容、媒体引用、是否重要等）。
- **媒体存储策略**：
  - 小体积图片可直接以 base64/blob 形式存入 IndexedDB；大体积图片建议控制压缩与数量，避免占用过多浏览器存储。
  - 语音可按 blob 存储，并在列表中仅保存引用键。
- **备份与导入导出（可选）**：
  - 提供「数据导出」能力（如导出为 JSON/压缩包）方便用户自行备份。
  - 提供「从文件导入」以恢复同一浏览器中的数据（注意覆盖/合并策略的提示）。

### 4.3 集成与接口

- 默认不依赖业务后端，应用可在纯静态托管环境下运行（如静态网站托管、GitHub Pages 等）。
- 如需后续扩展登录、云同步等功能，需在单独版本/阶段新增后端需求文档，此处不作约束。

### 4.4 安全与隐私

- 由于所有业务数据保存在本地浏览器，默认不上传到服务器，需在隐私说明中明确告知用户「数据仅存储在本机浏览器」。
- 若部署到 HTTPS 域名，仍建议全站使用 HTTPS，防止潜在中间人攻击及未来可能接入的网络请求风险。
- 避免在前端存储中保存不必要的敏感信息，如明文账号密码等。

### 4.5 性能与体验

- **资源**：首屏关键 CSS 内联或优先加载；图片懒加载、合适尺寸（如 srcset）；大图/语音按需加载。
- **指标**：TTI ≤ 3s（4G）；Lighthouse 移动端性能 ≥ 85。
- **可选 PWA**：Service Worker 缓存静态资源与部分 API 结果；manifest 提供图标、名称、主题色；「添加到主屏幕」后可全屏/独立窗口。

### 4.6 兼容与测试

- **浏览器**：iOS Safari 最近两代、Android Chrome 最近两代；必要时在真机与模拟器上测试。
- **分辨率**：以 375×667 为基准，适配到 428×926 等常见尺寸；横屏可做最小适配或提示竖屏使用。

## 五、风险与排期建议

### 5.1 分阶段交付

| 阶段 | 范围 | 说明 |
|------|------|------|
| **MVP** | 首页 + 爱的告白 + 恋爱日记（列表/新建/查看）+ 基础相册（上传/列表） | 移动端布局与触控达标，满足核心记录与浏览 |
| **v1.1** | 相册分类、封面、收藏与分享；里程碑的添加与时间线 | 完整 PRD 功能在移动端落地 |
| **v2.0（可选）** | 纪念日提醒、数据导出等 | 增强留存  |

### 5.2 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 弱网下首屏过慢 | 跳出率高 | 骨架屏、关键内容优先、接口分页与压缩 |
| 相册/相机权限被拒 | 无法上传 | 明确说明用途、提供「去设置开启」引导 |
| 部分机型 300ms 点击延迟 | 感觉不跟手 | 使用 `touch-action: manipulation` 或现有方案去除延迟 |
| 后端接口未就绪 | 前端阻塞 | 先用 Mock 数据与约定接口文档并行开发 |

## 六、验收标准汇总

- **功能**：实现 PRD 中首页、爱的告白、恋爱日记、爱的相册、相爱里程碑在移动端上的全部子功能点。
- **体验**：触控区域 ≥ 44×44px；首屏 TTI ≤ 3s（4G）；Lighthouse 性能 ≥ 85、可访问性 ≥ 90。
- **兼容**：iOS Safari、Android Chrome 最近两代主流版本；安全区域与竖屏适配。
- **安全与隐私**：HTTPS、敏感操作防护、私密/公开与权限说明明确。
